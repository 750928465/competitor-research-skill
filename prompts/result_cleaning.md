# 数据清洗提示词（灵活提取版）

## 任务

从搜索结果中提取有价值的信息，结构化为可用于后续分析的 JSON 数据。根据搜索内容的不同，灵活提取产品信息、技术架构、应用场景、发展趋势等内容。

## 核心原则

1. **根据实际内容灵活提取** - 不机械套用模板
2. **充分利用搜索结果** - 提取所有有价值的信息
3. **每个事实都标注来源**
4. **不确定的信息明确标注**
5. **信息有限时也要输出有价值的框架**

## 场景判断

根据搜索内容判断当前场景：

### 场景A：竞品/同类产品分析
- 搜索结果中有多款明确的产品
- 重点提取：产品信息、功能对比、定价等

### 场景B：新技术/概念研究
- 搜索结果主要是技术介绍、原理说明
- 重点提取：技术架构、核心概念、应用场景

### 场景C：单一产品/技术深度研究
- 搜索结果聚焦于某个产品或技术
- 重点提取：详细信息、技术细节、案例

### 场景D：趋势/市场研究
- 搜索结果主要是行业分析、趋势报道
- 重点提取：市场数据、发展趋势、机会挑战

## 提取指引

### 产品信息提取
- 从标题、摘要中识别产品名称
- 提取官网链接作为主要来源
- 收集功能描述、定价、目标用户等信息
- 如果没有明确产品，products 数组可以为空

### 技术信息提取
从搜索结果中提取：
- **技术架构**：系统组成、模块划分、技术栈
- **核心原理**：工作机制、底层原理
- **技术特点**：优势、局限性、适用范围
- **实现方式**：部署方式、集成方式、开发方式

### 应用场景提取
从搜索结果中提取：
- 典型应用场景
- 用户案例
- 行业应用
- 具体玩法

### 发展趋势提取
从搜索结果中提取：
- 市场趋势
- 技术演进方向
- 行业动态
- 机会与挑战

## 严格规则

1. **充分利用搜索结果** - 不要因为某些字段缺失就放弃提取
2. **灵活调整输出结构** - 根据搜索内容调整重点
3. **每个信息标注来源** - 包括 URL 或来源编号
4. **明确标记不确定信息** - 使用 "不明确"、"信息有限"
5. **避免编造** - 没有的信息不要虚构
6. **保持输出完整性** - 即使信息有限也要输出有价值的框架

## 特殊情况处理

### 搜索结果中没有明确产品
- 将 products 设为空数组 []
- 重点提取 tech_insights、use_cases、trends
- 输出技术分析框架

### 搜索结果信息有限
- 提取所有可用的片段信息
- 在 insights 中总结关键发现
- 标注 metadata 中各字段状态

### 搜索结果主要是技术文档
- 提取技术架构、原理、特点
- 收集应用场景和案例
- 输出技术深度分析框架
